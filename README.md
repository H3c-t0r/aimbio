# ‚ÑπÔ∏è About

Aim is an open-source developer framework that helps to build custom logging and observability layer for any AI system.

- Logging: Aim offers a unified interface for logging all kinds of data from your AI system, from end to end.
- Observability: Aim make it super easey to create a unified observability layer with a low-code Python interface.

## Why use Aim?

In a rapidly evolving AI landscape, Aim offers a robust and reliable way to log and monitor AI systems lifecycle:

- End-to-End Lineage: Aim enables you to capture and restore the end-to-end lineage of the entire AI system, providing clear visibility and traceability.
- Governance: Aim helps reduce risks associated with AI systems. Manage the system's lifecycle, resources, costs, vendors, etc by logging all moving pieces and monitoring the overall behavior.

## How does Aim work?

Aim is built around the following concepts:

- Aim Types: Python classes that allow you to define custom data types for logging. Log any data from anywhere.
- Aim Boards: Python files that help you create a unified observability layer using Aim's core toolkit. Build the UI you need and adapt as your needs evolve.
- Aim Packages: Aim Types and Boards are packaged and used as Python packages. You're just a step away from sharing your work with the community.

Aim's has the following main components:

- Aim Storage: A high-performing, RocksDB-based storage that is capable of storing 100,000s of log sequences.
- Aim Server: A powerful centralized server that collects logs from any node within the cluster.
- Aim Python SDK: The SDK includes building blocks of Aim Types and Boards.
- Aim UI: UI enables displaying boards, explorers for comparing the data and reports that enable to keep track of results and share progress.

## Open-source and community-driven

- Ecosystem: Aim's logging and observability components are packaged using Python, resulting in reusable modules. Users can easily implement these ready-made packages and contribute back to the community.
- Open-source & Self-hosted: Aim is fully open source and self-hosted, giving you full control and ownership of your data.

# üèÅ Quick start

Follow the steps below to get started with Aim.

## 1. Install Aim on your training environment

```shell
pip3 install aim
```

## 2. Run the Aim server and UI

```shell
aim server --package ml
```

```shell
aim ui --package ml
```

## 3. Integrate Aim with your code

```python
from aimstack.ml import Run

# Initialize a new run
run = Run()

# Log run hparameters
run["hparams"] = {
    "learning_rate": 0.001,
    "batch_size": 32,
}

# Log metrics
for i in range(10):
    run.track(i, name='loss', step=i, context={ "subset":"train" })
    run.track(i, name='acc', step=i, context={ "subset":"train" })
```

## Run the script and explore logs on the UI! üéâ

# üõ£Ô∏è Roadmap

- **Aim Actions:** Execute Aim-initiated actions. Actions could range from primitive automations to starting external workflows.
    - Attach external workflows to aim sessions (e.g. run a model checkpoint).
    - Implement arbitrary manipulations over aim objects (e.g. invert an image).
    - Importantly: Aim only connects the logged metadata to the action and initiates it. Aim does not provide the runtime.
- **Aim Callbacks:** Custom callbacks on storage-update.
    - Aim action executed on specific type of storage updates.
- **Aim Beats:** Notifications triggered on storage update.
    - A specific Aim action that triggers notification to various mediums (slack, email etc) when storage is updates and specific criteria is met.
- **Aim Logs Browser:** Logs discovery and management via Aim UI and SDK.
- **Aim JS SDK:** A JavaScript API specifically designed for developers to track logs from JavaScript applications.
- **Aim core kit++:**
    - **Interactive Visualization Components:** Render all needed UI components included as part of core Aim
    - **Higher-Level Visualization Components:** Avoid extra boilerplate codes using higher level building blocks. Potentially shipped as part of Aim Standard Package (ASP).
